# HKRR 理论

HKRR 来自影视飓风对爆款短视频的拆解研究，是判断一条视频能否留住人的四个核心要素。

## 四个要素

| 字母 | 含义 | 核心问题 |
|------|------|---------|
| **H** | Happiness（快乐） | 观众看完有没有笑、有没有轻松感、有没有惊喜？ |
| **K** | Knowledge（知识） | 观众学到了什么？能复述给别人吗？ |
| **R** | Resonance（共鸣） | 观众有没有感到"就是我"、"我也遇到过"？ |
| **R** | Rhythm（节奏） | 视频有没有快慢节奏？有没有让人喘气的地方？ |

## 使用规则

**K 是必选项。** 没有知识点的视频是娱乐，不是内容。

**每条视频至少满足 K + 任意一项。** 推荐组合：

| 组合 | 适用场景 | 特点 |
|------|---------|------|
| K + H | 技术科普、工具教程 | 幽默类比让技术不枯燥 |
| K + R | 投资、职场、人生哲学 | 戳中痛点让人有代入感 |
| K + R + H | 爆款内容标配 | 三者兼顾，留存率最高 |
| K + R + R(节奏) | 长内容、深度内容 | 节奏控制让人看完 |

## 在脚本中如何落地

### H（快乐）怎么做？

- **类比法**：把复杂概念比作生活中的东西
  - "RAG 就是让 AI 学会查资料——像开卷考试"
  - "区块链就是每个人都记账，没人能改"
- **反转法**：先建立预期，再打破它
  - "你以为 AI 会替代程序员……其实先替代的是……"
- **自嘲/梗**：用轻松语气讲严肃内容
  - "不是我说，这波操作真的绝"

### K（知识）怎么做？

- 一条视频只讲一个核心知识点
- 知识点要能用一句话概括（takeaway 句）
- 要有"框架"——让观众知道如何迁移到其他场景
- 举例要具体，不要说"比如 A 可以做 B"，要说"2022 年巴菲特用 X 方法买了 Y，赚了 Z"

### R（共鸣）怎么做？

- 开场用痛点唤醒（参考 hook-patterns.md）
- 中间加"你是不是也……"的旁白
- 结尾升华到更大的人文意义（可选，1 分钟视频不强求）

### R（节奏）怎么做？

- 快节奏场景（5-8秒）：Hook、认知冲突、数据冲击
- 慢节奏场景（12-20秒）：核心知识点解释、框架讲解
- 结尾场景（5-8秒）：极简 Payoff
- **规律**：快-慢-快-慢-快，像音乐节拍

## HKRR 自检清单

生成脚本后，逐项检查：

```
H（快乐）：
- [ ] 有没有幽默的类比或反转？
- [ ] 语气是否轻松，不像教科书？
- [ ] 至少有一处让人会心一笑？

K（知识）：
- [ ] 核心知识点是什么？（一句话写出来）
- [ ] 观众看完能复述这个知识吗？
- [ ] 有没有具体例子支撑？

R（共鸣）：
- [ ] 开场是否戳中目标观众的痛点？
- [ ] 有没有"你是不是也……"的代入感？
- [ ] 结尾有没有升华（哪怕一句）？

R（节奏）：
- [ ] 总时长是否控制在 60 秒内？
- [ ] 场景时长有没有快慢变化？
- [ ] 有没有超过 20 秒的单个场景（如有，考虑拆分）？
```

## 常见的 HKRR 失分项

| 问题 | 失分项 | 修复方法 |
|------|--------|---------|
| 全程严肃说教 | H 失分 | 加一个生活化类比 |
| 知识点太多（3个以上） | K 失分 | 取最重要的1个，其余删掉 |
| 没有目标观众感 | R 失分 | 开场加痛点唤醒 |
| 所有场景时长相同 | R（节奏）失分 | 调整时长，Hook 短、核心长 |
