# 视频质量评估标准（Quality Eval）

用于生成前预检和生成后评审两个阶段。总分 100 分，低于 70 分必须修复后再交付。

---

## 五个评估维度

### 1. 视觉重心度 (Visual Focus) — 30分

**核心问题**：画面是否有唯一的视觉中心？

| 分段 | 表现 |
|------|------|
| 0-10 | 满屏文字且字号接近；左右两边同时出现不相关动画 |
| 11-20 | 有主次但对比不够强，辅助信息抢眼 |
| 21-30 | 重点概念突出（≥96px），辅助信息通过透明度（≤0.5）或字号（≤主字1/3）降权 |

**自动判定规则**：
- 场景内最大字号 ÷ 最小字号 < 1.5 → 扣15分（字号层次不够）
- 同一场景有3个以上等权重元素 → 扣10分
- 主标题字号 < 80px → 扣10分

---

### 2. 动效生命力 (Dynamic Vitality) — 20分

**核心问题**：文字弹入后是否有持续的"微动"？

| 分段 | 表现 |
|------|------|
| 0-7 | 所有元素弹入后完全静止，像静态PPT |
| 8-14 | 有弹入动画但缺乏持续感 |
| 15-20 | 文字具有慢速悬浮（floating）或背景有轻微缩放（Ken Burns） |

**Spring 动画规范**：
- 弹出必须带轻微超越目标再回弹：`damping: 8-15`（bouncy/heavy 预设）
- 纯淡入没有质感，必须搭配 translateY 或 scale
- 静止后微动实现（可选，加分项）：

```tsx
// 慢速悬浮（Ken Burns / floating）
const floatY = interpolate(
  Math.sin((frame / fps) * Math.PI * 0.5),
  [-1, 1], [-6, 6]
);
// 背景轻微缩放
const bgScale = interpolate(frame, [0, durationInFrames], [1, 1.04], {
  extrapolateRight: 'clamp'
});
```

**分层出现 (Stagger) 规范**：
- 同屏多个元素必须错开，每个延迟 3-8 帧（100-250ms @ 30fps）
- 不能所有元素同时弹出
- 超过5个元素时，按脚本逻辑顺序依次出现

**视听同步（加分项）**：
- 关键词出现时机应对应口播重音位置
- 实现方式：在 script.json 的 voiceover 中用 `[accent:词]` 标记重音词，生成时匹配该词的出现帧

---

### 3. 视听一致性 (AV Sync) — 20分

**核心问题**：画面变化是否踩中音频关键词？

| 分段 | 表现 |
|------|------|
| 0-7 | 语音说完了动画还在播；或动画早结束语音还未开始 |
| 8-14 | 大致对齐，但有明显的 0.5秒以上偏差 |
| 15-20 | 关键词的变色/弹出与口播时间点对齐（误差 <10帧） |

**检查方法**：
- 场景时长（durationInFrames / fps）应 ≈ 口播字数 ÷ 3.5（字/秒）
- 口播 100 字 → 约 28秒 → 840帧
- 允许误差 ±20%，超出则拆分场景

**口播长度速查**：

| 口播字数 | 建议时长 | 帧数（30fps） |
|---------|---------|-------------|
| 20字 | 6s | 180f |
| 35字 | 10s | 300f |
| 50字 | 14s | 420f |
| 70字 | 20s | 600f |

---

### 4. 信息处理深度 (Information Density) — 15分

**核心问题**：是否对脚本进行了"视觉降噪"？

| 分段 | 表现 |
|------|------|
| 0-5 | 直接搬运脚本段落到屏幕，文字超过6行 |
| 6-10 | 有提炼但仍有冗余，某些词不在屏幕上出现 |
| 11-15 | 只展示核心 Token/关键词（≤5个/屏），复杂逻辑转化为图形连接 |

**一屏一重点原则**：
- 单场景独立词条 ≤ 7个（竖屏）
- 单行字符 ≤ 18字（竖屏）
- 单行字符 ≤ 28字（横屏）
- 信息超载时：宁愿增加场景数量，不要塞满一屏

**黄金比例布局**：
- 核心大字应占屏幕高度的 1/4
- 周围辅助小字字号 ≤ 大字的 1/3

---

### 5. 构图审美 (Composition) — 15分

**核心问题**：留白与比例是否合理？

| 分段 | 表现 |
|------|------|
| 0-5 | 文字贴边；布局过度对称死板；配色超过3种 |
| 6-10 | 有基本留白，配色基本规范 |
| 11-15 | 错落有致；配色严格 ≤3种（背景色+主色+强调色）；padding充足 |

**构图规范**：
- 安全 padding：竖屏 ≥ 80px，横屏 ≥ 60px 上下 / 120px 左右
- 配色上限：背景色 + 主文字色 + 1个强调色，禁止同时出现4种以上
- 布局要有错落：避免所有元素水平居中，可用左对齐+右侧留白

---

## 生成前预检（Pre-Check）

在生成任何场景代码前，对脚本逐项检查：

```
预检清单：
[ ] 每个场景是否只有1个核心重点？（维度4）
[ ] 主标题字号是否已设计为 ≥96px？（维度1）
[ ] 多个元素是否已设计 stagger 延迟？（维度2）
[ ] 场景时长是否与口播字数匹配？（维度3）
[ ] 配色是否控制在3种以内？（维度5）
[ ] 是否有超过6行文字的场景？→ 强制拆分（维度4）
```

发现问题**立即调整脚本/结构**，不要带着问题生成代码。

---

## 生成后评审（Post-Review）

生成代码后，逐场景打分：

```
评审表：
场景ID | 视觉重心 | 动效生命力 | 视听一致 | 信息密度 | 构图审美 | 总分
-------|---------|----------|---------|---------|---------|-----
scene_1|   /30  |    /20   |   /20  |   /15  |   /15  |  /100
scene_2|   /30  |    /20   |   /20  |   /15  |   /15  |  /100
```

**修复优先级**：
1. 维度1（视觉重心）— 最影响第一眼观感
2. 维度2（动效生命力）— 消除PPT感关键
3. 维度3（视听一致）— 影响渲染正确性
4. 维度4（信息密度）— 影响信息传达
5. 维度5（构图审美）— 加分项

**低于70分的场景必须修复，不得交付。**

---

## 常见问题 & 快速修复

| 问题 | 维度 | 快速修复 |
|------|------|---------|
| 所有文字同时出现 | 动效 | 给每个元素加 delay，每隔 5 帧一个 |
| 主标题太小（<80px） | 视觉重心 | 提升至 96-120px，删减文字换取空间 |
| 文字弹入后完全静止 | 动效 | 加 floating 微动：`Math.sin(frame/fps * Math.PI * 0.5) * 6` |
| 一屏超过6行文字 | 信息密度 | 只保留核心1-2句，其余移入口播不上屏 |
| 配色超过3种 | 构图 | 删除第4种颜色，降权为主色的透明变体 |
| 口播和画面明显不同步 | 视听 | 按字数公式重算 durationInFrames |
| 布局完全居中无变化 | 构图 | 改为左对齐 + 右侧留白，或上下错位排布 |
