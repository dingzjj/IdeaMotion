# 得到品控标准

得到的内容品控体系，核心是"感性材料优先"。把抽象概念变成可被感受的经验。

## 核心理念

**工具性内容 → 体验性故事**

| 错误做法 | 正确做法 |
|---------|---------|
| "RAG 的工作原理是检索增强生成" | "你问 AI 一个问题，它先去图书馆查资料，再回来回答你" |
| "价值投资需要分析基本面" | "巴菲特看一家公司，第一个问题是：10 年后它还在吗？" |
| "复利的威力很大" | "20 岁存 1 万，年化 10%，60 岁变 45 万。你的钱自己生钱。" |

## 挑战定义（必须先做）

在写任何内容前，必须定义清楚挑战：

```
让 [目标观众] 在 [场景] 下，[解决的问题]
```

**好的挑战定义示例：**
- 让**想用 AI 的产品经理**，在**选型时**，搞清楚 RAG 能干什么，不能干什么
- 让**加密货币新手**，在**投资前**，判断稳定币是否适合自己
- 让**职场新人**，在**第一次汇报**时，知道怎么结构化表达

**差的挑战定义：**
- 让所有人了解 RAG（太宽泛，没有场景）
- 让开发者学会 RAG（没有解决什么问题）

## 感性材料三类型（≥60%）

### 1. 具体案例
必须有时间、地点、人物、结果，不能是虚构的。

```
✅ "2011 年，巴菲特在金融危机后花 50 亿买入美国银行优先股，2018 年价值 270 亿。"
❌ "在某次金融危机后，有个投资者买入了银行股，赚了很多钱。"
```

### 2. 生活类比
把抽象概念映射到观众已有经验。

```
✅ "RAG 就像开卷考试——不是让 AI 死记硬背，而是让它带着书本来考试。"
✅ "区块链账本就像小时候全班同学都抄一份黑板上的通知，一个人改了没用，得所有人改。"
❌ "RAG 是一种检索增强生成方法，通过语义检索提升大语言模型的准确性。"
```

### 3. 数据/数字
必须有对比或场景感，孤立的数字没有力量。

```
✅ "每天多运动 30 分钟，10 年后患心脏病的概率降低 35%——相当于少抽了 10 年烟。"
✅ "ChatGPT 发布后 2 个月，用户突破 1 亿——Netflix 用了 3.5 年。"
❌ "运动对健康有好处，研究表明效果显著。"
```

## 品控三问（生成脚本后必答）

### 问 1：挑战是否清晰？
- 目标观众是具体的人，不是"所有人"
- 场景是具体的情境，不是"想了解"
- 问题是可执行的，看完能做某件事

### 问 2：感性材料是否 ≥60%？
统计脚本中：
- 案例数量：___
- 类比数量：___
- 有对比的数据：___
- 纯概念解释（减分项）：___

如果感性材料 <60%，强制增加类比或案例。

### 问 3：观众看完能否行动？
- 有没有一个"下次遇到这种情况，你可以……"的句子？
- 知识有没有迁移框架（不只是这个例子，能用到其他场景）？
- 结尾是否给了观众确定感（不是"你要思考一下"，而是"记住这一句话"）？

## 内容层次规范

### 不允许的表达
- "需要指出的是……"（书面语，改口语）
- "从多个维度来看……"（删掉，直接说维度是什么）
- "这是一个复杂的问题"（废话，删掉）
- "总的来说"（口播中避免，画面上不要出现）

### 推荐的口播风格
- 口语化："你有没有遇到过……""说白了就是……""说实话……"
- 有节奏：短句为主，最长不超过 20 字/句
- 有画面感：每句话说完，观众脑子里能浮现一个画面
- 有温度：像在和朋友聊天，不是在做 PPT 演讲

## 感性材料来源策略

当找不到真实案例时，按优先级：

1. **类比**（最容易创造）：找一个观众熟悉的生活场景，建立映射
2. **数据对比**：找权威数据，加上对比基准
3. **反例**：描述错误做法，让观众通过"不是这样"理解"应该是那样"
4. **虚构场景**（最后手段）：明确说"想象一下……"，不要伪装成真实案例
